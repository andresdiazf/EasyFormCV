# EasyFormCV — Project Context

## Overview
EasyFormCV automates job application form filling. Users upload a PDF CV, the system extracts profile data, detects form fields on a target URL, maps them, and fills them automatically via browser automation.

---

## Architecture

```
Frontend (React/Vite :5173) ──▶ API (Fastify/Node :3000) ──▶ pdf-parser (Python/FastAPI :8001)
                                                                └── browser-automation (Python/Playwright :8002)
                                                                └── PostgreSQL (Supabase via Prisma)
```

### Monorepo Structure

| Directory | Purpose |
|-----------|---------|
| `apps/api/` | Fastify REST API server (port 3000) |
| `apps/web/` | React SPA with Vite, TanStack Query, Tailwind (port 5173) |
| `packages/shared-schemas/` | Zod schemas + TypeScript types shared between API and frontend |
| `services/pdf-parser/` | Python FastAPI microservice for PDF text extraction (port 8001) |
| `services/browser-automation/` | Python FastAPI + Playwright microservice (port 8002) |
| `prisma/schema.prisma` | Canonical database schema (PostgreSQL) |

---

## Key Technical Decisions

- **Monorepo with npm workspaces**: Single `npm install`, shared config, `@easyformcv/*` aliases
- **Zod in shared-schemas**: Single source of truth for types & validation consumed by both API and frontend
- **Fastify over Express**: Faster, TypeScript-native, built-in schema validation
- **Python microservices**: PyMuPDF (PDF) and Playwright (browser automation) are Python-native
- **Prisma ORM v7**: Type-safe database access with PostgreSQL adapter
- **Graceful degradation**: If Python services are offline, API returns stub data with warnings
- **Offline caching**: Frontend caches GET responses in sessionStorage for offline resilience
- **HTTP timeouts**: All outbound requests use AbortController (30s pdf-parser, 180s browser-automation)
- **Login support**: browser-automation supports generic login flow before detect/run (Google, LinkedIn, etc.)
- **Security**: No stack trace leaks, path traversal prevention, file size limits, Zod input validation

---

## Database Models (Prisma)

- **Profile**: Singleton candidate profile (fullName, email, phone, location, summary)
- **FormMapping**: URL + detected fields + field-to-profile mappings
- **Run**: Automation execution records (status, filled/failed fields, screenshot path)

## API Endpoints

| Method | Route | Description |
|--------|-------|-------------|
| `POST` | `/api/v1/cv/upload` | Upload PDF, extract profile |
| `GET` | `/api/v1/profile` | Get latest profile |
| `PUT` | `/api/v1/profile` | Create/update profile |
| `POST` | `/api/v1/forms/detect` | Detect fields on a URL (optional auth) |
| `GET` | `/api/v1/forms/mapping` | Get saved mapping |
| `POST` | `/api/v1/forms/mapping` | Save field mapping |
| `POST` | `/api/v1/runs` | Start automation run |
| `GET` | `/api/v1/runs` | List recent runs |
| `GET` | `/api/v1/runs/:id` | Get run details |
| `GET` | `/health` | Health check |

## Common Workflows

### Add a new field to CandidateProfile
1. Add field to `CandidateProfileSchema` in `packages/shared-schemas/src/index.ts`
2. Add column to `model Profile` in `prisma/schema.prisma`
3. Run `npx prisma db push --accept-data-loss`
4. Update `PROFILE_FIELDS` constant in shared-schemas
5. Update `ProfileEditor.tsx` form fields
6. Update `CvUpload.tsx` display fields
7. Update `FormMapping.tsx` mapping dropdown (already uses PROFILE_FIELDS)
8. Update `pdf-parser/app/main.py` `CandidateProfile` model

### Add a new API route
1. Create file in `apps/api/src/routes/`
2. Add Zod schema in shared-schemas if request validation needed
3. Register in `apps/api/src/index.ts` with `app.register(route, { prefix: API_PREFIX })`
4. Add frontend function in `apps/web/src/lib/api.ts`

## Coding Conventions

- **TypeScript**: strict mode, ES2022 target, NodeNext module resolution
- **API routes**: Zod `.parse()` for input validation (no `as` type assertions)
- **Python**: FastAPI with Pydantic models, structlog for logging
- **Frontend**: Functional components with hooks, TanStack Query for data fetching
- **i18n**: All user-facing strings in `i18n.tsx` translations object
- **Error handling**: Catch errors in routes, log with `app.log`, return stub data on service failure
- **No secrets**: .env is gitignored, credentials only passed in memory at runtime
